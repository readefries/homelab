# ClouDNS Webhook Implementation - New Resources

## Summary

Added cert-manager DNS-01 support for ClouDNS via webhook provider. Ready to test on both floki and proxmox clusters.

## Files Created/Added

### Main Configuration
- **cert-manager/cloudns-issuers.yaml** — Secret + DNS-01 ClusterIssuers (staging + prod)
  - `cloudns-api-secret` (Secret) — ClouDNS sub-auth credentials (81154)
  - `letsencrypt-dns01-cloudns-staging` (ClusterIssuer) — Test issuance via Let's Encrypt staging
  - `letsencrypt-dns01-cloudns` (ClusterIssuer) — Production issuance

### Webhook Deployment
- **cert-manager/cloudns-webhook/chart/** — Vendored Helm chart (10 files)
  - Chart.yaml, values.yaml, .helmignore
  - templates/ (7 YAML templates)

- **cert-manager/cloudns-webhook/values-override.yaml** — Custom values
  - groupName: acme.xs4some.nl
  - fullnameOverride: cloudns-webhook
  - authIdType: sub-auth-id
  - secretName: cloudns-api-secret

- **cert-manager/cloudns-webhook/render.sh** — Templating script
  - Renders Helm chart → cloudns-webhook.yaml
  - Shows resource summary

- **cert-manager/cloudns-webhook/cloudns-webhook.yaml** — Generated manifests (13 resources)
  - ServiceAccount, ClusterRole × 3, ClusterRoleBinding × 3 (RBAC)
  - Deployment, Service (webhook)
  - APIService (Kubernetes API extension)
  - Certificate × 2, Issuer × 2 (TLS for webhook)

- **cert-manager/cloudns-webhook/deploy.sh** — One-command deployment
  - Interactive password entry
  - Automated kubectl apply
  - Rollout status wait

### Documentation
- **cert-manager/cloudns-webhook/README.md** — Quick-start guide
- **cert-manager/CLOUDNS-DNS01-SETUP.md** — Comprehensive setup & troubleshooting guide

## Quick Deploy

```bash
# Edit password in cloudns-issuers.yaml
vim cert-manager/cloudns-issuers.yaml

# Run deployment (interactive)
cd cert-manager/cloudns-webhook
./deploy.sh floki    # or: proxmox
```

Or use individual commands in CLOUDNS-DNS01-SETUP.md → "Deployment" section.

## What Gets Deployed

**Namespace:** cert-manager (existing, created if needed)

**Resources:**
1. Secret: `cloudns-api-secret` — ClouDNS API credentials (auth_id=81154, auth_password)
2. Deployment: `cloudns-webhook` — Webhook provider pod (ixoncloud/cert-manager-webhook-cloudns:1.1.1)
3. Service: `cloudns-webhook` — ClusterIP:443
4. APIService: `v1alpha1.acme.xs4some.nl` — Kubernetes API extension
5. ClusterIssuer: `letsencrypt-dns01-cloudns-staging` — Test ACME issuance
6. ClusterIssuer: `letsencrypt-dns01-cloudns` — Prod ACME issuance
7. RBAC: ServiceAccount, ClusterRoles, ClusterRoleBindings for webhook

**Configuration:**
- GroupName: `acme.xs4some.nl` (used in ClusterIssuer.spec.solvers[].dns01.webhook.groupName)
- CredentialsDir: `/creds` (mounted from secret)
- Webhook TLS: Auto-generated by cert-manager (uses self-signed CA + serving cert)

## Next Steps

1. **Edit password:** Update `auth_password` in cert-manager/cloudns-issuers.yaml
2. **Deploy:** Run `cert-manager/cloudns-webhook/deploy.sh floki` (or proxmox)
3. **Verify:** Check pod, apiservice, clusterissuer status (see CLOUDNS-DNS01-SETUP.md)
4. **Test:** Create test Ingress with `cert-manager.io/cluster-issuer: letsencrypt-dns01-cloudns-staging`
5. **Monitor:** Watch cert-manager and webhook logs for DNS challenge flow
6. **Transition:** Once working, update production Ingresses and disable Traefik ACME

## Files to Edit Before Deploy

```bash
vim cert-manager/cloudns-issuers.yaml
# Change line:   auth_password: "CHANGE_ME"

```

Alternatively, the deploy.sh script will prompt for this interactively.
